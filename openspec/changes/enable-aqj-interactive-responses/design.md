## Context
- 既存の AQ-J モバイルMOCは UI 比較に留まり、実際に 50 問へ回答して完了確認するフローが未整備。
- 今回は個人情報を一切扱わず、ブラウザ内の一時状態だけで回答・確認を完結させる。
- 回答保存は不要だが、全問回答後にサマリーを表示してテキストコピー等で持ち帰れるようにする。

## Goals / Non-Goals
- Goals:
  - 3 種フォームすべてで「質問 → バリデーション → 完了」のライフサイクルを統一した体験として提供。
  - ページを開いた瞬間に Q1 へアクセスできるようにする（導入フォーム無し）。
  - 未回答を自動検出し、該当箇所へスクロール/ジャンプできる UI を用意。
  - 完了状態で回答内容をリスト表示し、コピー/スクショしやすいフォーマットにまとめる。
- Non-Goals:
  - サーバ送信・永続保存・スコア計算・PDFエクスポートなどの仕組み。
  - 多人数/複数ユーザー向けの認証やID管理。

## Decisions
- Decision: 回答状態は `aqj.js` に共通ユーティリティを実装し、各フォームから import して制御する。
- Decision: サマリー表示はテーブル形式＋JSON風テキストを併記し、コピー操作を支援する。
- Decision: 未回答時の警告は常に質問番号を表示し、ドロップダウン版は該当セクションを自動で `open` にする、カルーセル/一問一答版は未回答スライドへ自動ジャンプする。
- Decision: アクセシビリティ確保のため、エラーメッセージや完了通知は `aria-live="polite"` な要素に流す。

## Risks / Trade-offs
- 50 問すべてを DOM で管理するため JS オブジェクトも 50 件を保持するが、静的ページとしては許容。
- 進捗サマリーや未回答ハイライトのロジックが複雑になる可能性 → 共通関数化で重複を抑える。

## Open Questions
- サマリーの形式（CSV/TSV/JSON 等）をどこまで提供するか。今回はテキスト2通り程度の最小限にとどめる。
- スマホでのコピー操作を補助する UI（例: 「回答をコピー」ボタン）をどこまで入れるか。
